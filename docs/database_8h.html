Internal method used to allocate a single value within a bucket.

This is an internal method, meaning it should not be used by any methods outside of database.h

Optionally, a pointer to char can be specified, to which the index to the ptbl_record for the corresponding bucket of the database_record.ptbl_record_tbl will be written.

unsigned char* database_kv_get_value ( Context_main *  ctx_main,
Record_database *  rec_database,
char *  ptbl_index,
unsigned long  k

Attempts to resolve the index of the kv_record specified by k to the region which the value component resides at in memory.

Optionally, a pointer to a char can be specified, to which the index of the ptbl_record for the bucket where key k resides will be written.

NOTE: Do not directly write to the region of memory that is returned by this function. Use database_kv_set_value() instead. How would you feel if you were trying to read some data, when some jackass comes along and overwrites it, leaving you with partially-written data?

Returns
A pointer to the value corresponding to the kv_record identified by k on success, or a 0 on failure

[out]ptbl_indexWhere the index of the found ptbl_record should be written

◆ database_ptbl_alloc()

unsigned char* database_ptbl_alloc ( Context_main *  ctx_main,
Record_database *  rec_database,
char *  ptbl_index,
int  page_count,
int  bucket 
)

Returns a pointer to the newly allocated region, of size main_context.system_page_size * bucket, on success, or 0 on failure.

Optionally, a pointer to a char can be specified, to which the index for the ptbl_record for the corresponding bucket of the database_record will be written.

The reason it's a char is because the maximum bucket number is 63. Obviously we don't need a lot of bits to represent that, actually we don't even need a full 8. :)

Returns
Pointer to a region of allocated memory on success, 0 on failure.
